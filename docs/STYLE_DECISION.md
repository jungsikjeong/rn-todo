# NativeWind vs StyleSheet

> 이 프로젝트는 복잡한 테마 시스템이 필요하지 않아서 **StyleSheet**를 선택했습니다.

## 고민

React Native에서 스타일 적용 방법으로 NativeWind와 StyleSheet 중 어떤 걸 선택해야 할지 고민했습니다.

NativeWind는 웹에서 Tailwind처럼 클래스네이밍으로 편하게 스타일 적용이 가능하다는 장점이 있지만, 실제 사용해보니 global.css와 tailwind.config.js 등 설정 파일이 필요하다는 번거로움이 있었습니다.

StyleSheet는 웹에서 Tailwind를 자주 쓰던 입장에서 스타일 적용이 좀 귀찮게 느껴졌습니다.

AI도 뾰족한 답변을 못 주는 거 같아서 직접 검색을 해봤는데, [Reddit](https://www.reddit.com/r/reactnative/comments/1er8kmn/is_nativewind_commonly_used_instead_of/)에서 "StyleSheet 자체가 이미 추상화인데 그 위에 NativeWind를 얹는 걸 선호하지 않는다"는 의견을 확인했고, 이 부분이 궁금해서 개발팀장님께 여쭤봤습니다.

## 결론 (개발팀장님 답변)

> 토씨하나도 빠트릴 내용이없이 모두 도움이 되었기에 그대로 원문을 복사했습니다.

실제로 NativeWind를 쓰면 추상화 레이어가 하나 더 끼어드는 것은 맞습니다. 이것 때문에 디버깅이 어려워지는 경우가 생길 수도 있습니다.

일반적으로 컴퓨터공학에서 추상화 레이어가 많은 것 자체로는 문제삼지 않는데요, 인터넷도 5개에서 7개의 추상화 계층을 깔아 두면서도 각 계층의 책임이 잘 분리되어 있으므로 수십년간 잘 작동하고 있습니다.

지금 이 케이스에서는 두 계층의 책임 분리가 모호하므로(둘의 목적과 역할에 겹치는 부분이 많으므로) 사용상 문제가 생길 수 있음을 지적하고 있는 것으로 생각합니다.

StyleSheet는 RN이 공식적으로 지원하는 API라는 장점이 있고 타입 유추도 잘 되는 등 단독으로 사용하기에 부족함이 없습니다. 웹에서는 css의 나사빠진 DX 때문에 고통받던 개발자들을 tailwind가 구원하는 그림이 나왔는데, RN에서는 상황이 조금 다르다고 하겠습니다.

NativeWind도 테마(dark:)나 반응형(md:) 등에서 나름의 장점은 있습니다. 하지만 tailwind가 동적인 값을 처리하지 못하고 애니메이션 처리가 곤란하다는 단점도 그대로 가져왔습니다. ( transform-x-[${xPos}] 같은 className이 작동하지 않는 문제 ) 이 경우 어차피 style prop에 StyleSheet를 넣어 써야 합니다.

그러면 NativeWind는 절대 못 써먹을 물건이냐 하면 그건 아닌데, 컬러 팔레트를 tailwind.config에 정의해 두고 bg-danger과 같이 쓰는 패턴은 여전히 편리합니다. 색상뿐 아니라 spacing, round 등등 앱 전체에 일관된 디자인을 적용하는 게 편해지는 부분은 tailwind 장점을 그대로 가져왔습니다.

## NativeWind가 적합한 프로젝트

아래 사항을 프로젝트가 많이 만족할수록 NativeWind는 사용할 만 합니다:

- 통일되고 합의된 테마/컬러팔레트/디자인시스템이 있다
- 라이트/다크/또는 그 외의 테마를 사용해야 한다
- 반응형 레이아웃을 지원해야 한다
- 디자인 관련 디버깅이 어려울 수 있다는 점을 받아들일 수 있다
- 복잡한 커스텀 애니메이션이 있다 (보통은 다른 라이브러리로 해결)

반대로, 위에 해당사항이 없거나 적은 프로젝트라면 이미 잘 만들어진 StyleSheet 사용을 고려해야 합니다. 웹에서는 특별히 tailwind가 필요하지 않은 상황에서도 끔찍한 css를 피하기 위해 tailwind를 설치해 쓰는 경우가 있으나 StyleSheet는 그다지 끔찍하지 않기 때문에 이런 저울질이 가능합니다.
